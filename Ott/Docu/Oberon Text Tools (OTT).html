Oberon Text Tools (OTT)

1. Introduction

OTT is a portable text i/o library for Oberon environments. It provides a stable, scalable API for manipulation of plaintext (streams of CHARs).

a) Goals

  • Simplicity. OTT is a scalable API that supports simple text i/o with a minimum of fuss, and complex i/o using multiple readers and writers.
  • Portability. OTT applications are intended to be portable to all OTT supported Oberon environments.
  • Reliability. OTT is intended to become a de-factor standard for I/O in Oberon systems.
OTT is not intended to support formatted text (font, colours, etc). UNICODE support may be added later. Binary data streams (8-bit byte streams) are supported by the lowest layer APIs in OTT.

b) OTT and OOC

OTT is based on the I/O sub-system of the OOC library and uses the channel/rider concept to support i/o to a variety of sources and destinations (texts, files, etc). A channel is a connection to some data location (i.e. a source or sink for data), and presents the data as a stream of bytes. Channels export Reader and Writer objects that can be used to access or modify the data in the channel. Since all Channel, Reader, and Writer types are derived from base abstract types, programs can be written to operate on any type of channel.

c) Three levels of APIs

OTT provides APIs with increasing flexibility:
  • OttIn and OttOut modules provide simple i/o to any channel.
  • The TextRider module provides more flexible scanning and formatting support, and supports converting native Oberon data types to and from plaintext. Alternately, the module Tokenizer provides basic string parsing similar to C's strtok() function.
  • The lowest level modules (eg. Files) provide byte-level access and multiple readers and writers.

d) OTT Channels

The types of channels supported are system-dependent. All OTT implementations offer Files. OTT on full "Oberon systems" such as Black Box or V4 will offer Texts (a channel that connects to the native Text object) and Log.

Since the channel types are not portable, it is recommended to separate the creation of channels from their use, which should be as OttChannel objects, or using OttTextRider.

2. Version Information

The current version of OTT is 0.1. This is the first release of OTT. It's based on OO2C-1.3.8.

It supports the following Oberon environments:

  • BlackBox version 1.3 (also known as Oberon/F)
Other versions (OO2C, V4, Sys3, etc) are to follow.

3. Documentation

OTT includes the following OOC library modules:
  • Ascii CharClass Strings ConvTypes LRealConv IntConv IntStr
  • Channel Texts Files Log TextRider In Out
Note. The OTT modules add an "Ott" prefix to the filename. For example, "OttAscii" corresponds to the OOC module "Ascii".
